设置WebView，使网页有权调用麦克风

重写WebChromeClient，授予网页权限。 当网页调用getUserMedia()获取权限时，会触发onPermissionRequest()，需要在方法中做一些处理。代码如下：

// 注意只有在Android 5.0以后才有这个方法
@RequiresApi(Build.VERSION_CODES.LOLLIPOP)
@Override
public void onPermissionRequest(PermissionRequest request) {
    // super.onPermissionRequest(request);

    if (request.getResources().length > 0 && request.getResources()[0].equals(PermissionRequest.RESOURCE_AUDIO_CAPTURE)) {
        Log.d("MainActivity", "申请麦克风权限");
        // 判断权限
        if (!lacksPermission(Manifest.permission.RECORD_AUDIO)) { // 已经拥有权限
            // 建议加入弹窗，让用户现在选择是否在这个网页使用麦克风
            request.grant(request.getResources()); // 直接同意授权
        } else {
            // 系统申请权限
            mPermissionRequest = request;
            ActivityCompat.requestPermissions(MainActivity.this,
                    new String[]{Manifest.permission.RECORD_AUDIO}, PERMISSION_REQUEST_CODE);
        }

    }
}

注意事项

  1. 网页必须是https，不然在请求权限的时候，会报SSL错误，触发WebViewClient的onReceivedSslError()方法。
  2. 取消必须由用户手动触发媒体播放mWebView.getSettings().setMediaPlaybackRequiresUserGesture(false);
